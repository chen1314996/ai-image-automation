核心能力

把重复操作收束成可控、可停、可追踪的生产流程

平台面向需要大量生成素材的创作流程：减少网页反复操作、减少手工保存、减少顺序混乱，让每一次批量输出都能回溯到参考图和提示词。

A

豆包 API 提示词

不再依赖豆包网页，直接通过大模型 API 基于参考图生成五组规整提示词。

L

Legil 自动生图

自动上传参考图、填入提示词、应用模型参数、等待生成并保存输出。

R

批量改尺寸

独立分页面控制 Legil，支持固定提示词、独立参数和独立输入输出目录。

S

稳定保存链路

优先保存原图，直链失败后尝试浏览器响应、页面下载，最后才截图兜底。

量产流程

从文件夹到最终图片，按名称排序、按序号输出

输入图按名称自然排序处理，输出文件名携带运行 ID、参考图序号、提示词序号和变体序号，打开文件夹就能看懂生成先后。

读取参考图

扫描输入文件夹，过滤支持的图片格式，按文件名自然排序。

生成提示词

调用豆包图文大模型 API，返回五组不同画面描述。

应用参数

在 Legil 图生图页面应用模型、比例、分辨率和输出数量。

等待完成

识别当前生成行，支持横版、竖版、方版和失败图跳过。

保存输出

打开大图后尽量保存原图，兜底截图，文件名保持可追踪。

改尺寸分页面

不走豆包，只控制 Legil 做批量比例调整

本地部署

双击启动，窗口常驻，按 Ctrl+C 停止

Windows 启动脚本会检查依赖、启动服务器并打开自动化平台。浏览器登录态保存在本地 `browser_data`，Legil 登录一次后可复用。

Windows_start.cmd port 3066 
1. 双击 Windows_start.cmd
2. 自动检查并安装依赖
3. 启动 Node.js 服务
4. 打开 http://localhost:3066/
5. 在终端按 Ctrl+C 停止服务器
