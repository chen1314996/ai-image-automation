# 项目总览

这个项目是一个本地运行的 AI 生图自动化平台。它的核心目标是把“参考图生成提示词，再用提示词批量生成图片”的流程自动化，减少人工反复复制、粘贴、等待和保存图片。

## 一句话说明

把本地文件夹里的参考图交给豆包生成提示词，再把提示词交给 Legil 生成图片，最后把生成结果保存到本地输出文件夹。

## 主要流程

1. 前端页面打开本地服务。
2. 用户配置输入文件夹、输出文件夹、浏览器模式和生成参数。
3. 后端读取输入文件夹里的参考图。
4. 豆包接口根据参考图生成多组提示词。
5. 工作流控制器把提示词逐条送到 Legil。
6. Playwright 控制浏览器完成上传参考图、填写提示词、点击生成、等待结果。
7. Legil 自动化模块保存生成图片。
8. 前端通过实时日志看到执行进度。
9. 飞书相关模块可以查看状态、接收通知或远程控制任务。

## 当前启动方式

启动方式保持不变：

```bash
npm start
```

或者：

```bash
node server.js
```

浏览器访问地址保持不变：

```text
http://localhost:3066
```

## 第一轮安全重构做了什么

第一轮只整理后台入口和接口目录，不改变核心业务流程。

保留不变：

- `npm start`
- `node server.js`
- 端口默认值 `3066`
- 所有 `/api/...` 接口地址
- `browser_data` 登录状态目录
- `automation-config.json` 配置读取方式
- `automation-secrets.json` 密钥读取方式
- 豆包、Legil、完整工作流的核心自动化逻辑
- 前端页面文件 `public/index.html`

新增结构：

```text
src/
  app.js
  server/start.js
  routes/*.routes.js
```

根目录的 `server.js` 现在只是兼容入口，它会继续启动真正的后台服务。
