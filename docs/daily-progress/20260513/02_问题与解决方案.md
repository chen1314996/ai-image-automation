# 问题与解决方案 - 2026-05-13

## 问题 1：飞书点击继续任务后提示没有任务

### 现象
用户在飞书点击“继续任务”，机器人回复没有任务，但自动化平台页面明明有可继续的创意拓展任务。

### 原因
飞书继续任务最初只检查完整量产工作流的恢复状态，没有同时检查创意拓展的恢复状态。

### 解决方案
- 统一状态查询来源。
- 将创意拓展恢复状态纳入飞书“进度”和“继续任务”判断。
- 继续任务优先匹配当前可恢复任务。

### 结果
飞书进度和自动化平台页面状态趋于一致，创意拓展停止后可从飞书继续。

## 问题 2：飞书卡片按钮太多且文字展示不全

### 现象
卡片按钮过多，移动端或窄窗口中文字被截断。

### 解决方案
- 精简主卡片按钮。
- 只保留 6 个最高频操作。
- 其他操作改用文字指令。

### 结果
卡片更清晰，误触和展示截断概率降低。

---

## 问题 3：飞书按钮会弹出浏览器窗口

### 现象
点击继续任务、状态等飞书按钮后，会打开浏览器窗口或黑色页面。

### 原因
旧卡片使用 URL 按钮，飞书客户端会把本地回调链接交给系统默认浏览器打开。

### 解决方案
- 新版卡片改为 `value` 回调按钮。
- 通过飞书 SDK 长连接处理卡片动作。
- 重新发送新版后台回调卡片。

### 额外说明
旧卡片已经发在飞书聊天里，本地代码无法修改旧消息的按钮结构，因此需要使用新卡片。

## 问题 4：偶发弹出类似终端的新窗口

### 现象
用户反馈改完后仍然时不时弹一下，像新开了一个终端窗口。

### 排查结果
Windows 计划任务每分钟运行：

```text
powershell.exe -NoProfile -WindowStyle Hidden -ExecutionPolicy Bypass -Command ...
```

即使带 `-WindowStyle Hidden`，交互式计划任务仍可能短暂弹出 Windows Terminal 或控制台。

### 解决方案
- 新增 `scripts/start-watchdog-hidden.vbs`。
- 计划任务改为：

```text
wscript.exe C:\Users\dd\Desktop\项目\scripts\start-watchdog-hidden.vbs
```

- VBS 内部以隐藏窗口方式启动 Watchdog。

### 结果
计划任务 `Last Result: 0`，不再使用 PowerShell 作为计划任务入口。

## 问题 5：主服务掉线时无法通知

### 原因
如果 `server.js` 已经崩掉，主服务内部通知逻辑也会一起消失。

### 解决方案
- 新增独立 `feishu-watchdog.js`。
- Watchdog 与主服务分离。
- 主服务掉线时 Watchdog 仍可通知飞书并自动重启。

### 验证
强制停止 `server.js` 后，Watchdog 自动重启成功。

## 问题 6：Legil 卡住或异常时缺少现场证据

### 现象
Legil 生成失败时，只能看到日志，无法知道页面状态。

### 解决方案
- 异常时自动截图。
- 飞书发送异常说明和截图。
- 截图保存到 `runtime/legil-error-screenshots/`。

### 结果
后续排查可以直接查看现场截图。

## 问题 7：连续失败仍继续消耗任务

### 风险
如果登录失效、页面异常、参数错误，连续失败时继续跑会浪费时间和额度。

### 解决方案
- 连续失败达到阈值后暂停。
- 默认阈值为 3 次。
- 飞书通知用户等待确认。
- 创意拓展和完整工作流尽量保留恢复状态。

---

## 问题 8：Legil 保存图片命名不够可追踪

### 现象
保存文件名里出现“提示词1”这类泛化内容，批量图片较多时不方便回溯来源。

### 解决方案
- 从提示词对象或提示词文本中提取简短标题。
- 清理“提示词”“画面提示词”“prompt”等泛化前缀。
- 输出文件名保留运行流水、参考图序号、提示词序号、变体序号和简短标题。

### 结果
量产页面保存图片时能保留对应的简短提示词标题，排序和回溯都更清晰。

## 问题 9：Legil 输出 4 张时经常只检测到 3/4

### 现象
日志显示：

```text
已检测到 3/4 张新图，继续等待其余输出...
等待图片生成超时
```

截图显示 Legil 页面横向区域中已出现 3 张图，第 4 张可能被横向裁切、隐藏或页面结构未被扫描到。

### 不采用的方案
不采用“等待超时后只要发现已有图就强行当成功保存”的兜底，因为这会影响所有 Legil 流程，也不符合用户希望“跟创意拓展页面一样”的方向。

### 解决方案
- 保留严格超时错误逻辑，不在超时后兜底。
- 新增“稳定输出”判定：页面回到可生成状态，且新图连续两次扫描结果一致。
- 量产、改尺寸、创意拓展的批量任务都启用同一套判定。
- 保存阶段仍由统一的 `saveGeneratedImages()` 负责扫描新输出并保存。

### 结果
当 Legil 页面实际上已经结束生成，但只稳定检测到部分有效输出时，可以像创意拓展页面一样进入保存流程，避免无意义等到超时中断。
