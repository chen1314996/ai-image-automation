### 2. 飞书卡片按钮优化
**问题**

飞书按钮过多时文字展示不全；旧版 URL 按钮点击后会打开浏览器窗口，用户不希望飞书控制触发弹窗。

**优化**

- 精简为 6 个主按钮：状态、进度、开始量产、继续任务、停止任务、重启服务。
- 更多功能保留文字指令入口。
- 新版卡片改为后台回调按钮，不再使用 URL 按钮。
- 卡片底部提示“新版后台回调按钮，不会打开浏览器”。

**验证**

- 新版卡片 JSON 不包含 `url`。
- 新版卡片 JSON 不包含 `multi_url`。
- `/api/health` 中 `cardActionReady=true`。
- 飞书测试消息发送成功。

### 4. 飞书异常通知系统
**新增模块**

- `feishu-notification-service.js`
- `health-monitor.js`

**通知场景**

- 工作流停止。
- 工作流异常中断。

- 创意拓展停止、完成、异常。

**默认策略**

### 5. Legil 异常截图与自动恢复
**用户要求**

Legil 页面异常时自动保存截图，并通过飞书发送截图。

**实现内容**

- Legil 执行失败时保存全页截图。
- 等待生成超时时先截图。
- 通过飞书发送异常说明和截图。
- 截图保存目录：`runtime/legil-error-screenshots/`。

**自动恢复策略**

- Legil 页面丢失：自动重开。
- Legil 卡住：自动刷新一次并重试。
- 连续失败 3 次：暂停任务等待确认。

### 6. 独立 Watchdog 与自动重启
**目标**

主服务 `server.js` 掉线时，仍能由外部进程发现、通知飞书并自动重启。

**实现内容**

- 新增 `feishu-watchdog.js` 独立进程。
- 每 30 秒检查 `/api/health`。
- 连续 2 次失败后通知飞书。
- 自动尝试重启 `server.js`。
- 服务恢复后发送恢复通知。
- 记录 `lastDownAt`、`lastRestartAt`、`lastRecoveryAt`。

**真实演练**

- 强制停止 `server.js`。
- 不手动启动。
- Watchdog 自动检测掉线。
- Watchdog 自动拉起新的 `server.js`。
- 健康接口恢复 200。

### 10. Legil 检测与保存逻辑对齐
**问题**

Legil 输出数量为 4 时，页面可能只稳定显示 3 张，导致日志出现“已检测到 3/4 张新图，继续等待其余输出”，最后等到超时报错并中断。

**用户确认方向**

不要简单在超时后强行保存已有图片，而是看创意拓展页面的 Legil 检测和保存逻辑，量产和改尺寸跟它一致。

**完成内容**

- 撤回“超时后凑合保存已有图”的兜底方向。
- 在 Legil 等待器中增加“页面已空闲且新图稳定”的判定。
- 量产、改尺寸、创意拓展的批量 Legil 任务都启用同一套稳定输出判定。
- 如果页面生成状态已经结束、按钮恢复可用、新图连续两次扫描一致，则进入保存流程。
- 保存流程继续使用同一套新输出图扫描和打开大图保存逻辑。
