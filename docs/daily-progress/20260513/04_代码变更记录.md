## 详细变更记录

### 1. `feishu-cli-bridge.js`

**新增能力**

- 使用飞书 SDK 消费消息事件。
- 支持卡片按钮回调。
- 长连接异常时记录状态。
- 支持文本消息发送。
- 支持控制卡片发送。
- 支持图片发送，供 Legil 异常截图通知使用。

### 2. `feishu-card-builder.js`

**控制卡片**

- 状态。
- 进度。
- 开始量产。
- 继续任务。
- 停止任务。
- 重启服务。

**优化**

- 不再使用 URL 按钮。
- 使用 `value` 触发后台回调。
- 底部提示新版后台回调按钮不会打开浏览器。

### 3. `feishu-command-router.js`

**支持的文字指令**

- 帮助。
- 状态。
- 进度。
- 日志。
- 浏览器状态。
- 开始量产。
- 停止/继续工作流。
- 停止/继续创意拓展。
- 重启工作流或服务器。

### 4. `feishu-control-service.js`

**职责**

把飞书按钮或文字指令映射为本地 API 调用。

**示例**

- `start_mass` 调用 `/api/workflow/start`。
- `continue_workflow` 调用 `/api/workflow/resume` 或创意拓展恢复接口。
- `stop_workflow` 调用 `/api/workflow/stop` 或 Legil 停止接口。
- `restart_server` 调用 `/api/server/restart`。

### 5. `feishu-notification-service.js`

**能力**

- 通知冷却。
- 通知历史。
- 级别区分：信息、警告、错误、严重。
- 异步发送失败不影响主任务。
- 支持截图路径，先发文本再发图片。

### 6. `health-monitor.js`

**能力**

- 定时采集健康快照。
- 检测任务长时间无进展。
- 检测飞书连接状态变化。
- 启动时通知服务状态和可继续任务。

### 7. `feishu-watchdog.js`

**能力**

- 检查 `/api/health`。
- 连续失败后发飞书掉线通知。
- 自动重启 `server.js`。
- 恢复后发飞书恢复通知。
- 防止重复运行。
- 写入 `runtime/feishu-watchdog.status.json`。

### 8. `scripts/register-watchdog-task.ps1`

**初版**

使用 PowerShell 启动 Watchdog。

**问题**

交互式计划任务会偶发弹出终端窗口。

**修复**

改为 `wscript.exe` 启动 `start-watchdog-hidden.vbs`。

### 9. `scripts/start-watchdog-hidden.vbs`

**作用**

通过 `shell.Run command, 0, False` 隐藏启动 Watchdog，避免终端窗口闪现。

### 10. `server.js`

**新增健康与通知接口**

- `/api/health`
- `/api/notifications/status`
- `/api/health/test-notification`
- `/api/config/notifications`
- `/api/watchdog/status`
- `/api/watchdog/start`

**新增任务通知**

- 工作流完成/停止/异常通知。
- 创意拓展完成/停止/异常通知。
- 改尺寸完成/停止/异常通知。
- Agent 完成/失败/取消通知。

**新增通知配置**

- `feishuEnabled`
- `taskCompletionEnabled`
- `serverStartupEnabled`
- `staleProgressEnabled`
- `staleThresholdMinutes`
- `notificationCooldownMinutes`
- `legilScreenshotEnabled`
- `autoRecoveryEnabled`
- `pauseOnConsecutiveFailures`
- `consecutiveFailureThreshold`
- `watchdogAutoRestartEnabled`

### 11. `workflow-controller.js`

**恢复与暂停**

- 新增连续失败统计。
- 连续失败达到阈值后暂停。
- 保存恢复快照。
- 恢复快照包含运行模式和 Legil 生成参数。

**Legil 命名**

- 新增提示词标题提取。
- 保存文件名追加简短提示词标题。
- 清理“提示词1”这类泛化命名。

**Legil 检测**

- 量产任务传入 `acceptStablePartialOutputs`，与创意拓展/改尺寸保持同一套稳定输出判定。

### 12. `src/routes/legil.routes.js`

**改尺寸**

- 支持有头/无头模式。
- 支持 Legil 生成参数。
- 接入连续失败暂停和异常通知。
- 接入稳定输出判定。

**创意拓展**

- 支持恢复状态。
- 支持停止后继续任务或新任务。
- 支持无参考图时跳过上传。
- 接入稳定输出判定。

### 13. `src/services/legil/`

**拆分职责**

- `generation-flow.js`：Legil 主生成流程。
- `generation-settings.js`：模型、比例、分辨率、输出数量设置。
- `output/generation-waiter.js`：生成等待与稳定输出判定。
- `output/output-scanner.js`：新输出图与失败占位扫描。
- `save/save-flow.js`：批量保存输出图片。
- `save/preview.js`：打开大图和截图兜底。
- `save/download.js`：图片下载和文件校验。
- `config-methods.js`：保存目录、参考图目录、文件命名。

**关键修复**

- 撤掉“超时后只要有图就保存”的全局兜底。
- 新增“页面已空闲且新图稳定”判定。
- 保存流程继续只保存本轮新输出，避免拿到历史图或参考图。

### 14. `public/index.html` 与 `public/js/`

**页面能力**

- 量产、改尺寸、创意拓展运行模式选择。
- 通知配置面板。
- 创意拓展停止后可继续任务或新任务。
- 提示词预览按方向标题展示，勾选方向时选择其下提示词组。

**前端拆分**

- `public/js/workflow.js`
- `public/js/resize-workflow.js`
- `public/js/creative-workflow.js`
- `public/js/config-panels.js`
- `public/js/platform-config.js`
- `public/js/creative-prompts.js`
