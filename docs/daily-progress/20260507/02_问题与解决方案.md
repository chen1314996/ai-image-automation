### 问题1: Legil 平台图片保存失败
**现象:**
```
[22:34:49] 保存图片失败: 不支持的图片格式，请手动保存
[22:42:52] 保存图片失败: 不支持的图片格式，请手动保存
[22:52:54] 保存图片失败: 不支持的图片格式，请手动保存
[22:58:51] 保存图片失败: 不支持的图片格式，请手动保存
```

**原因分析:**
1. Legil 平台的图片地址是相对路径 `/legil/_next/image?url=...`，不是完整 HTTP URL
2. 原始下载逻辑只支持 `http://` 开头的地址
3. 右键菜单点击不稳定，浏览器下载行为不一致

**解决方案（3次迭代）:**

**迭代1 - 拼接完整URL:**
```javascript
// 处理相对路径
if (imgSrc.startsWith('/')) {
    const pageUrl = page.url();
    const baseUrl = new URL(pageUrl).origin;
    fullUrl = baseUrl + imgSrc;
}
```

**迭代2 - 增加截图备选:**
```javascript
// 方法2: 使用 Playwright 截图功能
const box = await imgElement.boundingBox();
if (box && box.width > 100 && box.height > 100) {
    await imgElement.screenshot({ path: savePath });
}
```

**迭代3 - 简化保存逻辑:**
```javascript
// 最终方案：先截图，失败再下载
async saveGeneratedImage(page, index, generationStartTime) {
    // 方法1：先尝试截图（最可靠的方式）
    try {
        await imgElement.screenshot({ path: savePath });
        return savePath;
    } catch (screenshotError) {
        // 方法2：直接下载
        return await this.downloadImageDirectly(page, imgElement, savePath);
    }
}
```

**状态:** 已解决 ✅

### 问题2: 豆包提示词提取不完整
**现象:**
```
[22:43:57] ⚠️ 只找到 1 组提示词，少于预期的5组
[22:43:57] ❌ 只提取到 1 组提示词，请检查回复格式
```

**原因分析:**
1. 豆包的回复格式使用带空格的"第 1 组"格式
2. 提示词内容放在代码块（```plaintext）中
3. 原正则表达式 `/第[一二三四五12345]组/` 无法匹配带空格的格式

**解决方案:**
```javascript
// 更新正则表达式，支持空格
/groupPattern = /第\s*([一二三四五12345])\s*组[：:]?\s*([^\n]*)([\s\S]*?)(?=第\s*[一二三四五12345]\s*组[：:]|$)/g

// 提取代码块内容
const codeBlockMatch = content.match(/```(?:plaintext)?\s*([\s\S]*?)```/);
if (codeBlockMatch) {
    content = codeBlockMatch[1].trim();
}

// 合并画面内容和风格要求
fullPrompt = `${title}\n${content}`;
```

**状态:** 已解决 ✅

### 问题3: 图片生成检测过早触发
**现象:**
```
[22:34:49] ✅ 检测到 3 张生成图片 (3秒)
```
用户反馈: "legil平台保存图片需要等3-5分钟"

**原因分析:**
1. 原逻辑只检测页面图片数量
2. 检测到了历史图片而非新生成的图片
3. 没有等待生成过程完成

**解决方案:**
```javascript
async waitForImageGeneration(page) {
    // 第一步：等待"生成中..."出现（说明开始生成了）
    let generatingStarted = false;
    while (!generatingStarted && generatingCheckCount < 30) {
        const generatingText = await page.$('text=生成中');
        if (generatingText && await generatingText.isVisible()) {
            generatingStarted = true;
            break;
        }
    }

    // 第二步：等待"生成中..."消失（说明生成完成了）
    while (waited < maxWaitTime) {
        const generatingText = await page.$('text=生成中');
        if (!generatingText) {
            // "生成中..."消失了，生成完成
            return true;
        }
        await browserController.sleep(checkInterval);
    }
}
```

**状态:** 已解决 ✅

### 问题4: 浏览器锁定文件导致启动失败
**现象:**
```
无法打开 Legil 页面
检测到已存在的浏览器实例，先关闭...
```

**原因分析:**
- Playwright 使用 `launchPersistentContext` 时会在 `browser_data` 目录创建 `SingletonLock` 文件
- 异常退出时锁定文件未清理，导致下次启动失败

**解决方案:**
```javascript
// 启动前清理锁定文件
try {
    const lockFile = path.join(USER_DATA_DIR, 'SingletonLock');
    if (fs.existsSync(lockFile)) {
        fs.unlinkSync(lockFile);
        console.log('已清理锁定文件');
    }
} catch (e) {
    // 忽略清理错误
}
```

**状态:** 已解决 ✅

## 经验总结

1. **图片保存策略**: 截图比下载更可靠，应作为首选方案
2. **正则表达式**: 处理中文文本时要考虑空格和全角字符
3. **生成检测**: 检测过程状态（生成中）比检测结果更准确
4. **浏览器自动化**: 持久化上下文需要处理锁定文件清理
