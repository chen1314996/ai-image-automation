# 问题与解决方案 - 2026-05-10

## 今日遇到的问题

### 问题1：豆包网页自动化维护成本高

**现象：**
- 豆包网页回复格式不稳定
- 需要等待网页回复、识别 DOM、解析代码块和纯文本
- 每张图还需要新建对话窗口，流程慢且脆弱

**解决方案：**
- 改为调用火山方舟豆包图文大模型 API
- 本地图片直接作为 API 输入
- API 返回后统一解析为 5 组提示词数组
- 工作流不再依赖豆包网页

---

### 问题2：API Key 不能泄露

**风险：**
用户提供 API Key 后，如果写入 Git 跟踪文件，会造成泄露。

**解决方案：**
- 敏感配置放入 `automation-secrets.json`
- `.gitignore` 忽略 `automation-secrets.json`、`.env`、`.env.*`
- 文档只说明填写位置，不记录真实 Key

---

### 问题3：改尺寸页面保存配置报错

**现象：**
点击“保存改尺寸配置”时提示错误。

**根因：**
前端和后端对改尺寸配置结构不一致，生成参数没有独立归入 `resize.generationSettings`。

**解决方案：**
- 后端增加 `normalizeResizeConfigPayload()`
- 改尺寸配置统一包含 `inputFolder`、`outputFolder`、`promptTemplate`、`generationSettings`
- 前端保存时传入独立的 `config.resizeLegilGeneration`

---

### 问题4：改尺寸参数影响量产参数

**现象：**
改尺寸页面切换宽高比、分辨率、输出数量后，可能影响量产页面。

**根因：**
两个页面一度共用了同一份 Legil generation 配置。

**解决方案：**
- 量产: `config.legilGeneration`
- 改尺寸: `config.resizeLegilGeneration`
- 后端运行改尺寸任务时临时设置，任务结束后恢复原量产参数

---

### 问题5：改尺寸页面检测不到 Legil 生成完成

**现象：**
Legil 平台已完成图片生成，但后台一直显示等待。

**根因：**
改尺寸页面一开始没有完全复用量产页面成熟的检测与保存逻辑。

**解决方案：**
- 改尺寸调用同一套 `legilAutomation.generateImage()`
- 检测、失败图跳过、保存下载均复用量产核心逻辑
- 仅输入图路径、输出目录、提示词模板、生成参数独立

---

### 问题6：横版/竖版图片无法识别生成完成

**现象：**
- 方图检测正常
- 横版和竖版一直等待

**根因：**
横版/竖版显示在黑底容器里，`img` 自身尺寸不满足旧阈值。

**解决方案：**
- 识别输出槽位而非只识别图片元素
- 使用槽位宽高作为完成判断依据
- 保存时通过 `data-src`、`srcset` 等字段定位真实输出图

---

### 问题7：大图保存有时退化为截图

**现象：**
同一批图中，有的能直接保存大图，有的直链 HTTP 403。

**根因：**
不同输出图的 OSS/CDN 授权状态不同，部分原图地址不允许后端直接请求。

**解决方案：**
- 收集更多候选大图地址
- 优先去缩略参数获取原图
- 增加浏览器真实打开图片响应下载
- 页面内下载失败后才截图

---

### 问题8：Legil 登录态不能持久保存

**现象：**
关闭带 Legil 的浏览器窗口后重新打开，需要重新登录。

**根因：**
`storage_state.json` 与 `browser_data` 混用，旧状态可能覆盖新登录态；并且关闭浏览器窗口后后端没有及时重置状态。

**解决方案：**
- 只使用 `browser_data` 持久化 profile
- 取消启动时导入 `storage_state.json`
- 恢复早期可用的 `browser_data_backup_20260508_154344`
- 监听浏览器 close/disconnected 事件

---

### 问题9：打开 Legil 变卡

**现象：**
打开 Legil 后页面明显变慢。

**根因：**
打开阶段新增了模型检测和二次切换逻辑，导致额外等待页面控件和操作下拉菜单。

**解决方案：**
- 撤销打开后强制切换模型逻辑
- 打开 Legil 只负责打开页面
- 默认模型交给 Legil 的浏览器 profile 记忆
- 真正生成时再按量产/改尺寸各自参数应用

---

## 预防措施

1. **打开页面与自动化操作分离**
   - 打开网页只负责打开
   - 参数应用放在真正生成前

2. **配置隔离**
   - 量产和改尺寸配置必须使用不同字段
   - 临时运行参数不能污染全局配置

3. **敏感信息安全**
   - API Key 永不写入 Markdown、Git 跟踪文件或日志
   - 只记录配置方式，不记录真实值

4. **检测逻辑以页面实际结构为准**
   - 横版/竖版不能套用方图阈值
   - 识别输出槽位比识别单个 `img` 更稳
