# Web应用程序循环QA测试流程文档

## 测试目标
验证完整自动化流程：豆包生成提示词 → Legil生成图片 → 全自动循环处理

## 测试模式
- **浏览器模式**: 始终使用带界面浏览器（headless: false）
- **循环测试**: 持续运行直到所有功能验证OK
- **实时监控**: 通过截图查看运行状态

## 阶段三：全流程功能测试（核心）

### 3.1 单张参考图完整流程

**步骤1: 配置准备**
- 设置参考图文件夹（包含1张测试图）
- 设置Legil参考图文件夹
- 设置输出文件夹
- 点击"确认"按钮

**步骤2: 打开浏览器**
- 点击"打开网站"打开豆包和Legil
- ⚠️ **需要用户手动操作**: 登录豆包账号
- ⚠️ **需要用户手动操作**: 登录Legil账号

**步骤3: 启动工作流**
- 点击"一键启动完整工作流"
- 确认对话框

**步骤4: 验证豆包流程**
- [ ] 自动上传参考图到豆包
- [ ] 自动发送提示词
- [ ] 等待回复（1-3分钟）
- [ ] 成功提取5组提示词
- [ ] 显示在日志中

**步骤5: 验证Legil流程**
- [ ] 自动切换到Legil页面
- [ ] 自动上传参考图（如配置了Legil参考图）
- [ ] 自动填入第1组提示词
- [ ] 点击生成按钮
- [ ] 等待生成完成（3-5分钟）
- [ ] 自动保存图片到输出文件夹
- [ ] 重复5次（5组提示词各生成1张）

**步骤6: 验证完成**
- [ ] 显示"本轮工作已完成"弹窗
- [ ] 输出文件夹中有5张生成的图片
- [ ] 日志显示处理完成

### 3.2 多张参考图循环测试
- [ ] 配置文件夹包含3张参考图
- [ ] 启动工作流
- [ ] 验证第一张图处理完成
- [ ] 验证自动处理第二张图
- [ ] 验证自动处理第三张图
- [ ] 验证最终生成15张图片（3张×5组）

## 需要用户协助的位置

### 必须手动操作（因安全限制）
1. **登录豆包**: 输入账号密码/验证码
2. **登录Legil**: 输入账号密码
3. **人机验证**: 如有验证码需要手动完成

### 可选手动操作
1. **选择参考图**: 可手动指定特定图片
2. **调整提示词**: 可在豆包回复后修改
3. **中断流程**: 可随时停止工作流

---

## 测试完成标准

当满足以下条件时，测试完成：

✅ **功能完整**
- 豆包自动化：上传→发送→提取提示词 全部OK
- Legil自动化：上传参考图→填入提示词→生成→保存 全部OK
- 循环处理：多张参考图自动循环处理 OK

✅ **稳定性**
- 连续3次全流程测试无错误
- 无内存泄漏
- 浏览器可正常开关

✅ **结果验证**
- 输出文件夹中有正确数量的图片
- 图片质量符合预期
- 日志记录完整

---

## 开始测试

测试人员准备就绪后，执行：
```bash
node tests/cycle/start-cycle-test.js
```

测试将持续循环执行，直到所有功能验证通过或手动停止。
