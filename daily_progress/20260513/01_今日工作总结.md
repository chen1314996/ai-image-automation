### 2. 飞书卡片按钮优化
**问题：**
飞书按钮太多，文字显示不全，并且旧版 URL 按钮会打开浏览器窗口。

**优化：**
- 精简为 6 个主按钮：
  - 状态。
  - 进度。
  - 开始量产。
  - 继续任务。
  - 停止任务。
  - 重启服务。
- 更多功能保留文字指令。
- 新版卡片改为后台回调，不使用 URL 按钮。
- 卡片底部提示“新版后台回调按钮，不会打开浏览器”。

**验证：**
- 新版卡片 JSON 不包含 `url`。
- 新版卡片 JSON 不包含 `multi_url`。
- `cardActionReady=true`。

### 4. 飞书异常通知系统
**新增模块：**

**通知场景：**

**默认策略：**

### 5. Legil 异常截图与自动恢复
**用户要求：**
Legil 页面异常时自动保存截图，飞书发送截图。

**实现内容：**
- Legil 执行失败时保存全页截图。
- 等待生成超时时先截图。
- 通过飞书发送异常说明和截图。
- 截图保存目录：`runtime/legil-error-screenshots/`

**自动恢复策略：**
- Legil 页面丢失：自动重开。
- Legil 卡住：自动刷新一次并重试。
- 连续失败 3 次：暂停任务等待确认。

---

### 6. 独立 Watchdog 与自动重启
**目标：**
主服务 `server.js` 掉线时，仍能由外部监控发现并通知/重启。

**实现内容：**
- 新增 `feishu-watchdog.js` 独立进程。
- 每 30 秒检查 `/api/health`。
- 连续 2 次失败后通知飞书。
- 自动尝试重启 `server.js`。
- 服务恢复后发送恢复通知。
- 记录 `lastDownAt`、`lastRestartAt`、`lastRecoveryAt`。

**真实演练：**
- 强制停止 `server.js`。
- 不手动启动。
- Watchdog 自动检测掉线。
- Watchdog 自动拉起新的 `server.js`。
- 健康接口恢复 200。
