# 问题与解决方案 - 2026-05-13

## 问题 1：飞书按钮点击后提示没有任务

### 现象
用户在飞书点击“继续任务”，机器人回复没有任务，但自动化平台页面明明有可继续任务。

### 原因
飞书继续任务最初只检查完整工作流恢复状态，没有同时检查创意拓展恢复状态。

### 解决方案
- 统一状态查询来源。
- 将创意拓展恢复状态纳入飞书“进度/继续任务”判断。
- 继续任务优先匹配当前可恢复任务。

### 结果
飞书进度和自动化平台页面状态趋于一致。

## 问题 2：飞书卡片按钮太多且文字展示不全

### 现象
卡片按钮过多，移动端或窄窗口文字被截断。

### 解决方案
- 精简主卡片按钮。
- 只保留最常用 6 个。
- 其他操作改用文字指令。

### 结果
卡片更清晰，误触概率降低。

---

## 问题 3：飞书按钮弹出浏览器窗口

### 现象
点击继续任务等按钮后，会打开一个浏览器窗口或黑色页面。

### 原因
旧卡片使用 URL 按钮，飞书客户端会交给系统默认浏览器打开本机回调链接。

### 解决方案
- 新版卡片改为 `value` 回调按钮。
- 通过飞书 SDK 长连接处理卡片动作。
- 重新发送新版后台回调卡片。

### 额外说明
旧卡片已经发到飞书聊天里，本地代码无法修改旧消息的按钮结构，因此需要使用新卡片。

## 问题 4：时不时弹出类似终端的新窗口

### 现象
用户反馈改完后仍时不时弹一下，像新开了一个终端窗口。

### 排查结果
Windows 计划任务每分钟运行：
```text
powershell.exe -NoProfile -WindowStyle Hidden -ExecutionPolicy Bypass -Command ...
```
即使带 `-WindowStyle Hidden`，交互式计划任务仍可能短暂弹出 Windows Terminal/控制台。

### 解决方案
- 新增 `scripts/start-watchdog-hidden.vbs`。
- 计划任务改为：
```text
wscript.exe C:\Users\dd\Desktop\项目\scripts\start-watchdog-hidden.vbs
```
- VBS 内部使用隐藏窗口启动 Watchdog。

### 结果
跨整分钟触发后，计划任务 `Last Result: 0`，不再使用 PowerShell 作为计划任务入口。

## 问题 5：主服务掉线时无法通知

### 原因
如果 `server.js` 已经崩掉，主服务内部通知逻辑也会一起消失。

### 解决方案
- 新增独立 `feishu-watchdog.js`。
- Watchdog 与主服务分离。
- 主服务掉线时 Watchdog 仍可通知飞书并自动重启。

### 验证
强制停止 `server.js` 后，Watchdog 自动重启成功。

## 问题 6：Legil 卡住或异常时缺少现场证据

### 现象
Legil 生成失败时，只能看到日志，无法知道页面状态。

### 解决方案
- 异常时自动截图。
- 飞书发送异常说明和截图。
- 截图保存在 `runtime/legil-error-screenshots/`。

### 结果
后续排查可以直接看现场截图。

---

## 问题 7：连续失败仍继续消耗任务

### 风险
如果登录失效、页面异常、参数错误，连续失败时继续跑会浪费时间和额度。

### 解决方案
- 连续失败达到阈值后暂停。
- 默认阈值为 3 次。
- 飞书通知用户等待确认。
- 创意拓展和完整工作流尽量保留恢复状态。
