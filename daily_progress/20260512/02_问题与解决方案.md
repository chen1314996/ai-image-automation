# 问题与解决方案 - 2026-05-12

## 问题 1：项目 Agent 输出质量不足

### 现象
项目内 Agent 生成表格内容比 ChatGPT 调用 Agent 生成表格粗略很多。

### 根因判断
- 项目传给 Agent 的上下文不够完整。
- 原始表格内容没有充分结构化。
- 输出格式要求不够严格。
- 没有二次检查“每个方向 5 组提示词”。
- 没有质量评分机制。

### 解决方案
- 解析原始输入表格，将字段结构化。
- 提示词中明确要求方向标题、方向说明和 5 组画面提示词。
- 增加质量报告，识别缺字段、数量不足、内容过短。
- 对低质量结果进行二次补全。

### 后续目标
达到 ChatGPT 调用 Agent 的详细程度和方向丰富度。

## 问题 2：服务器重启后创意拓展任务丢失

### 现象
用户希望工作流程被意外打断或重启服务器后，仍能记住之前任务状态。

### 原因
运行中状态如果只在内存中维护，重启后会丢失。

### 解决方案
- 将创意拓展恢复状态写入本地配置。
- 重启时如果发现 `queued/running/stopping`，转为 `interrupted`。
- 页面加载时查询恢复信息。
- 继续任务时只提交剩余提示词。

### 结果
重启后可选择继续之前创意拓展任务。

## 问题 3：提示词预览内容过多

### 现象
提示词预览展示所有提示词，页面信息过载。

### 用户期望
只展示方向标题，勾选一个方向即选择该方向下 5 组提示词。

### 解决方案
- 将提示词按方向分组。
- UI 只渲染方向标题和选择框。
- 生成时展开为方向下的 5 组提示词逐一执行。

### 结果
页面更适合批量创意选择。

## 问题 4：用户不清楚如何通过飞书控制平台

### 现象
用户提出通过飞书企业自建应用和机器人控制平台，但需要先规划。

### 解决方案
规划为三层：
1. 飞书企业自建应用机器人负责收发消息。
2. 后端指令路由负责理解“状态/进度/停止/继续”等命令。
3. 本机控制服务调用现有自动化接口。

### 补充
可使用 `lark-cli` 做配置和调试，但不能复用已有项目的机器人配置。

## 问题 5：飞书 CLI 机器人已有其他项目在用

### 风险
复用旧 profile 或旧机器人会造成：
- 指令误触发其他项目。
- 消息发送到错误会话。
- 权限和回调混乱。

### 解决方案
- 创建新的企业自建应用。
- 应用名：`AI生图自动化平台`。
- 使用独立 profile。
- 使用独立白名单和通知 chat_id。
