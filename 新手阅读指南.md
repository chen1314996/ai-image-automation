# 新手阅读指南

这份文件放在根目录，方便你随时打开。  
你不需要先懂 Node.js、Express、Playwright，只要按下面的顺序看，就能知道这个项目大概怎么跑。

## 这个项目做什么

这个项目是一个 AI 生图自动化工具，主要做四件事：

1. 从本地文件夹读取参考图。
2. 调用豆包/火山方舟 API 生成提示词。
3. 把提示词和参考图交给 Legil。
4. 等 Legil 生成图片后，自动保存到本地输出文件夹。

网页只是一个操作面板，真正的自动化逻辑在后台 Node.js 代码里。

## 启动项目

```bash
npm start
```

或者：

```bash
node server.js
```

默认访问地址：

```text
http://localhost:3066
```

`server.js` 只是兼容入口，真正的启动代码在 `src/server/start.js`。

## 目录怎么理解

```text
src/
```

后台新结构。后台启动、接口路由、Legil 拆分后的模块都在这里。

```text
src/server/
```

后台启动相关：

- `start.js`：启动入口，只负责串起来。
- `context.js`：后台接口共用的配置、状态和工具函数。
- `routes.js`：统一注册所有接口模块。
- `runtime-services.js`：启动飞书 CLI、健康检查、守护监控。
- `shutdown.js`：关闭服务时保存状态、停止后台服务。
- `startup-banner.js`：控制台启动提示。

```text
src/routes/
```

后台接口，每个文件对应一组功能。比如：

- `workflow.routes.js`：完整工作流。
- `legil.routes.js`：Legil 生成、停止、改尺寸、创意续跑。
- `config.routes.js`：页面配置保存和读取。
- `feishu.routes.js`：飞书通知和远程控制。

```text
src/services/legil/
```

Legil 自动化拆分后的代码。  
如果你想看 Legil 是怎么保存图片的，从 `image-save.js` 看起；如果想看怎么识别生成结果，从 `output-detection.js` 看起。

```text
public/
```

前端页面：

- `index.html`：页面结构和按钮。
- `js/`：按钮点击后做什么。
- `css/`：页面样式。

```text
tests/
```

测试脚本、测试数据、测试截图和历史测试报告。

```text
docs/
```

历史文档、旧测试计划、归档材料。你平时优先看根目录这份指南和 `代码地图.md`。

## 按需求找代码

### 我想看“点开始完整工作流”发生什么

1. 先看 `public/js/workflow.js`，这里负责前端按钮点击。
2. 再看 `src/routes/workflow.routes.js`，这里接收前端请求。
3. 最后看 `workflow-controller.js`，这里真正串起豆包和 Legil。

### 我想看“Legil 怎么保存图片”

1. 先看 `src/services/legil/image-save.js`，它是保存入口。
2. 再看 `src/services/legil/save/download.js`，这里负责下载。
3. 再看 `src/services/legil/save/preview.js`，这里负责截图兜底。
4. 最后看 `src/services/legil/save/save-flow.js`，这里负责保存主流程。

### 我想看“Legil 怎么判断图片生成好了”

1. 先看 `src/services/legil/output-detection.js`。
2. 再看 `src/services/legil/output/generation-waiter.js`。
3. 再看 `src/services/legil/output/output-scanner.js`。

### 我想看“某个前端按钮对应哪个文件”

打开：

```text
public/js/README.md
```

里面有按钮、JS 文件、后台接口的对应表。

## 不建议随便改的东西

- `browser_data/`：浏览器登录状态。
- `automation-config.json`：你的运行配置。
- `automation-secrets.json`：密钥配置。
- `storage_state.json`：登录态相关文件。
- 真实输入/输出文件夹：可能是你正在使用的生产目录。

这次重构没有主动清理这些文件。
